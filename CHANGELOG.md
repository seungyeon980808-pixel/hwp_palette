## [미출시]

`v1.3.0` 이후 누적된 변경. **릴리즈할 때 이 절의 내용을 버전 번호와 함께 아래로
옮긴다.** (버전 번호는 아직 정하지 않았다 — `main.py`의 `VERSION`은 `1.3.0`
그대로다.)

> 규모로 보면 라이브러리·팔레트·마크다운 변환이 통째로 들어왔으므로
> **`v1.4.0` 이상**이 적절해 보인다. 판단은 릴리즈 시점에.

### 추가

- **커스텀 팔레트** — 탭 + 블럭 4종(문자/템플릿/서식 조합/양식). 격자에서 드래그로
  자유 배치, 블럭이 많아지면 세로 스크롤.
- **개인 라이브러리** — 서식·문자·템플릿·양식 4종을 등록해 재사용.
  한글에서 만든 결과물을 통째로 캡처하므로 병합 표·이미지 든 표도 안 깨진다.
- **마크다운 변환(Ctrl+T)의 `\라벨\` 문법** — 등록한 항목을 이름으로 호출.
  템플릿 속 빈칸 `\` 에 아랫줄 내용이 위에서부터 채워진다(`-` 한 줄이면 건너뜀).
- **양식(.hwp 파일 통째) 등록** — 파일만 고르면 등록되고, 클릭하면 새 문서로
  열린다. 용지·여백·머리말까지 원본 그대로. *(개선안 26)*
- **서식 적용 문법 `\명령{내용}`** — 줄 일부에만 서식 적용. 구조는 LaTeX 에서
  빌려왔다(`\textbf{...}`). *(개선안 27)*

  ```
  다음 중 \굵게{옳지 않은} 것은?
  \굵게\기울임\크기15{나는 이제 할 수 있는게 없다.}
  ```

  명령을 원하는 만큼 앞에 쌓으면 모두 적용된다(순서 무관 — LaTeX 의 누적 규칙).
  `\굵게` `\기울임` `\밑줄` `\15`(크기) `\자간-5` `\색빨강` `\함초롬바탕`
  `\글꼴나눔고딕`, 그리고 라이브러리에 등록해 둔 서식 라벨을 그대로 쓸 수 있다.
  `{ }` 안에는 `\라벨\` 호출도, 빈칸 `\` 도, 다른 서식(중첩)도 넣을 수 있다.
  글자 그대로의 역슬래시는 `\\`.
- **라이브러리 내보내기/가져오기** — 항목 목록과 조각 .hwp 를 zip 하나로 묶어
  동료와 주고받는다. 언제나 추가만 하고 덮어쓰지 않는다. *(개선안 30)*
- **어절 단위 줄바꿈** 기능 블럭 — 한글 기본값(글자 단위)에서 어절 단위로.
  *(개선안 28)*
- **앱 로그(`app.log`)** — 조용히 삼켜지던 실패를 기록. *(개선안 4)*
- **블럭 이름 툴팁** — 이름이 잘렸을 때만 표시. *(개선안 15)*
- **실행 취소 안내** — 되돌리기는 한글의 Ctrl+Z로 한다는 안내. *(개선안 16)*
- **단위 테스트 58개** — `python -m unittest discover -s tests`.
  한글 없이 실행 가능한 파싱·저장소 계층을 덮는다. *(개선안 22)*

- **양식 채우기 (AI와 주고받기)** — 양식 파일을 던지면 채울 자리를 뽑아 클립보드에
  넣어준다. Claude 같은 AI에 붙여넣고 채운 답을 다시 붙여넣으면 **원본 서식 그대로**
  채워진 문서가 한글에서 열린다.

  **AI 호출은 프로그램이 하지 않는다** — 사람이 복사·붙여넣기로 나른다. 그래서
  API 키도, 네트워크도, 비용도 없다.

  원리: HWPX 는 zip + XML 이고 글자는 `<hp:t>` 태그에 들어 있다. 그 태그 내용만
  바꾸고 나머지 바이트는 그대로 다시 쓴다 → 표·병합·글꼴·이미지가 살아남는다.
  실측(2026-07-19): 이미지 5개·셀 10개·병합 모두 보존, 한글에서 정상 열림.
  `.hwp`(바이너리)는 직접 고칠 수 없어 한글을 통해 HWPX 로 바꾼 뒤 처리한다.

  한계: **글자만 갈아끼운다.** 행 추가 같은 표 구조 변경은 안 된다.

- **사진 폴더 연결 — `\사진이름\` 으로 그림 삽입.** 라이브러리 창에서 폴더를
  한 번 연결해두면, 문서에 `\실험사진1\` 이라고 쓰고 변환할 때 그 폴더의
  `실험사진1.png` 가 글자처럼 그 자리에 들어간다(문서에 포함되므로 원본을
  지워도 그림은 남음). 등록 절차 없이 파일 이름이 곧 라벨이다.
  확장자는 안 써도 되고(png/jpg/jpeg/gif/bmp/tif/webp), 하위 폴더는 뒤지지
  않는다. 라벨 우선순위: 등록 항목 > 내장 문자 > 사진 폴더.
  실측(2026-07-19): 글 사이 삽입·서식 구간 안 삽입 모두 정상.

- **환경설정 격자에 '＋ 줄 추가' 버튼.** 빈 줄을 늘려 놓을 자리를 만들고,
  **줄이 늘면 창 자체가 그만큼 커진다**(줄이면 다시 줄어든다). 예전엔 높이 260px
  캔버스 안에서 스크롤해야 했다.
- **환경설정에서 빈칸을 끌어 새 블럭 만들기.** 격자의 빈칸을 한 칸 누르거나
  두세 칸 끌어 **칸 수를 먼저 정하면**, 무엇을 넣을지 고르는 창이 이어서 열린다.
  예전에는 '+ 문자 / + 템플릿 …' 버튼으로 추가한 뒤 칸 수를 따로 바꿔야 했다.
  (기존 추가 버튼도 그대로 남아 있다 — 빠른 경로)

- **블럭이 격자 위의 사각형이 됨 — 가로·세로 크기를 자유롭게.** 예전에는 블럭이
  목록 순서대로 흘러가는 구조라 **세로로 큰 블럭을 만들 수 없었다**(줄을 건너뛴다는
  개념 자체가 없었다). 이제 각 블럭이 `(row, col)` 좌표에서 `span×rows` 칸을
  차지한다. 기존 배치는 보던 그대로 좌표로 옮겨진다(일회 이전).
  - 환경설정에서 빈칸을 **가로·세로로 끌면** 그 사각형 크기로 새 블럭을 만든다
  - 선택 블럭을 `↔＋ ↔－ ↕＋ ↕－` 로 한 칸씩 늘리고 줄인다
  - 타일을 **빈칸에 놓으면 그 자리로 정확히 이동**한다(예전엔 맨 뒤로만 갔다)
- **칸 크기가 칸 수에 맞춰 변한다(적응형).** 칸을 늘리면 칸이 작아져 격자 폭은
  그대로 유지된다 → 오른쪽에 빈 공간이 안 생긴다. 기본 가로 **15칸**(칸 23px).
- **환경설정에 `＋ 칸` / `－ 칸` 버튼.** 칸을 줄일 때 오른쪽 끝에 블럭이 있으면
  잘리지 않도록 막고 안내한다.

  - **메인 버튼칸 높이 동기화** — 버튼칸이 여러 줄이면 변환 버튼도 같은 높이로
    늘어난다. 픽셀 계산 대신 grid `sticky="ns"` 로 Tk 가 맞추게 했다.

### 변경

- 팔레트의 **"기능" → "서식 조합"** 으로 표기 변경. 라이브러리의 "서식"(문서에서
  캡처)과 구분하기 위함. 저장 형식(`type: "function"`)은 그대로라 기존 설정이
  그대로 동작한다. *(개선안 10)*
- **엔진 3분할** — `hwp_engine`(코어: 연결·문서·글꼴·표·찾기) /
  `exam_engine`(시험문제 조판) / `engine_library`(라이브러리·팔레트·변환 실행).
  *(개선안 19, 20)*
- **실패 보고 통일** — `report_error()` 하나로 app.log + 메시지박스 + 상태표시줄에
  동시에 남긴다. 예전에는 "오류"라는 제목만 떠서 무슨 작업이 실패했는지도 알 수
  없었다. *(개선안 12)*
- **config.json 키 소유권 명시** — `settings.CONFIG_KEY_OWNERS` 표.
  소유 모듈이 아닌 곳에서 남의 키를 직접 만지지 않는다. *(개선안 21)*
- **지역 import 규칙** — 순환 참조 회피와 플랫폼 의존성 두 경우만 허용.
  *(개선안 24)*
- 흩어져 있던 매직넘버를 이름 붙은 상수로 승격. *(개선안 23)*
- `insert_file` 의 `keep_*` 인자를 전부 명시 — 값은 그대로지만 무엇이 딸려오는지
  코드에 드러나게 했다. *(개선안 6)*

### 수정

- **템플릿 아래 사용자가 쓴 `\` 가 함께 지워지던 문제.** `strip_slot_markers` 가
  삽입 지점부터 **문서 끝까지** 훑었다. 삽입 범위 안으로 제한.
  조용히 일어나고 인쇄 전에는 알 수 없는 훼손이라 위험도가 높았다. *(개선안 5)*
- **양식 블럭이 메인 팔레트에서만 다른 모양으로 보이던 문제.** 환경설정에서는
  📄+연녹색인데 메인에서는 ƒ+흰 배경이었다. *(개선안 31)*
- **같은 라벨을 쓰는 항목이 조용히 무시되던 문제.** 이름은 유일성을 보장하는데
  라벨은 아무도 검사하지 않아, 두 번째 항목은 `\라벨\` 로 영영 호출되지 않으면서
  아무 표시도 없었다. 이제 등록·수정 시 겹치면 알려준다. *(개선안 3)*
- **양식 라벨을 줄 중간에 쓰면 "등록되지 않은 라벨"이라고 잘못 안내**하던 문제.
- 라이브러리에서 삭제해도 팔레트 블럭이 고아로 남던 문제. *(개선안 1)*
- 템플릿 이름을 바꾸면 팔레트 블럭 연결이 끊기던 문제 — id 참조로 전환.
  *(개선안 2)*
- 한글 창이 최대화돼 있을 때 변환하면 창이 줄어들던 문제 — 창 배치를 저장했다
  복원. 클래스명 대소문자 오타로 창을 못 찾던 버그도 함께 수정.
- 마크다운 변환 시 한글의 대화상자 처리 모드가 영구히 바뀌던 문제 —
  `pyhwpx.find()` 대신 `RepeatFind` 직접 사용.
- 표 폭 설정이 반영되지 않던 문제(`WidthType`), 셀 병합 후 다음 표가 셀 안에
  중첩되던 문제.

### 수정 (자체 리뷰에서 발견 — 위 기능들을 만들면서 같이 들어간 것들)

- **서식이 뒤따르는 글 전체를 물들이던 문제.** 한글은 새 글자에 앞 글자의
  서식을 물려주므로 `\굵게{중요} 사항입니다` 에서 '사항입니다'와 그 뒤 모든 줄까지
  굵게 나왔다. 줄 시작 시점의 서식을 떠 뒀다가 감싸지 않은 구간에 되입힌다.
- **양식 변환이 함께 선택된 다른 줄을 삼키던 문제.** 양식은 새 문서를 여는데,
  선택 영역은 이미 지워진 뒤라 나머지 내용이 갈 곳 없이 사라졌다. 이제 지우기
  **전에** 감지해서 막고 안내한다.
- **표 안에서 빈칸이 하나도 안 채워지던 문제.** `GetPos()` 의 문단 번호는 표
  안에서 셀 기준으로 다시 세어지는데, 본문 기준 번호와 더해 범위를 계산했다.
- **표 안의 사용자 `\` 가 여전히 지워질 수 있던 문제.** 위와 같은 이유로 문단
  범위 검사가 표를 걸러내지 못했다. 템플릿이 선언한 **빈칸 개수**를 상한으로
  함께 걸어 막았다.
- 가져오기에서 조각 파일이 없어 건너뛴 항목이 이름·라벨을 선점해, 뒤따르는
  멀쩡한 항목이 있지도 않은 충돌로 이름이 바뀌던 문제.

### 수정 (창 깜빡임)

- **변환을 누르면 한글 창이 작아졌다 다시 커지던 문제.** pyhwpx 의 `Hwp()`
  생성자가 `XHwpWindows.Active_XHwpWindow.Visible = visible` 을 실행하는데,
  이 대입이 최대화된 창을 보통 크기로 되돌린다(실측: 최대 1094x1934 →
  보통 1080x802).

  예전에는 창 배치를 저장했다 복원하는 방식으로 막으려 했지만, 그건 '되돌리기'라
  **깜빡임 자체는 남는다.** 이제 한글이 이미 떠 있으면 **생성자를 아예 거치지 않고**
  붙는다(실행 중인 COM 객체에 직접 부착). 한글이 꺼져 있을 때만 `Hwp()` 로
  새로 실행하는데, 그때는 보존할 최대화 상태가 없어 문제가 되지 않는다.

  연결 생존 확인도 바꿨다 — pyhwpx 의 `hwp.Version` 은 문자열을 파싱하는
  프로퍼티라 표기가 다르면 살아 있는 연결도 '죽었다'고 오판할 수 있다.
  파싱하지 않는 원시 COM 값을 보도록 했다.

### 수정 (실사용에서 나온 것)

- **이름 미리보기가 한 글자 뒤처져 보이던 문제 (2차 수정).** 1차 수정
  (키를 뗄 때 다시 그리기)으로는 부족했다 — 조합 중인 글자는 키를 떼도 여전히
  위젯에 없기 때문. Windows IMM API(ImmGetCompositionStringW)로 **지금 조합
  중인 글자를 IME 에 직접 물어봐서** 위젯 값 뒤에 붙여 그린다. 이제 '기본문항'을
  치는 동안 미리보기도 실시간으로 따라온다. 조합 문자열은 'ㅎ→하→항' 같은
  중간 상태를 지나므로 표시에만 쓰고, 저장은 여전히 조합 확정 후 위젯 값을 쓴다.
- (1차 수정 기록) **이름 미리보기가 한 글자 뒤처져 보이던 문제.** '기본문'을 치는데 미리보기는
  `\기본\`로 나왔다. 한글 IME 로 **조합 중인 마지막 글자**는 화면에만 그려질 뿐
  위젯에는 아직 안 들어와 있기 때문이다(코드 순서 문제가 아님 — 위젯에 실제로
  들어간 글자는 즉시 반영되는 것을 확인). 이제 키를 뗄 때와 포커스가 빠질 때도
  다시 그리고, **저장 직전에 조합을 확정**시켜 마지막 글자가 빠지지 않게 했다.
  같은 문제가 있던 문자 등록·서식 조합 창도 함께 고쳤다.
- **템플릿 캡처 시 `[WinError 32] 다른 프로세스가 파일을 사용 중` 오류.**
  조각을 임시 이름(`_tmp_*.hwp`)으로 저장한 뒤 uuid 이름으로 **바꾸는** 구조가
  원인이었다. 한글은 캡처 과정에서 그 파일을 문서로 열어 붙들 때가 있고,
  **한 번 연 파일은 문서를 닫아도 잠금을 놓지 않는다**(실측). 그래서 이름 바꾸기가
  터졌다 — 사용자가 뭘 잘못한 게 아니라 구조 자체가 문제였다.
  → **처음부터 최종 이름으로 저장**하게 바꿨다. 바꿀 일이 없으니 오류가 원천적으로
  사라진다(최종 이름으로 바로 저장하면 한글이 그 파일을 열지도 않는 것을 확인).
  구버전이 남긴 `_tmp_*.hwp` 찌꺼기와 한글에 열린 채 남은 문서는 자동으로 청소한다.
  ※ 직전에 "기다렸다 재시도" 방식으로 고쳤다고 했으나, 닫아도 잠금이 안 풀리는
  것을 실측하고 접근을 바꿨다.

- **복잡한 문서에서 표 하나만 캡처했는데 다른 내용까지 다 들어가던 문제.**
  표를 **개체로 선택**한 상태(SelectionMode 4 — 표 테두리 클릭, 자동 선택 등)에서
  `FileSaveBlock_S` 가 선택만 저장하지 않고 **문서 전체를 저장**했다(실측).
  그 비대한 조각을 삽입하면 한글이 버벅거리다 멈추는 것까지가 한 세트였다.
  → 캡처를 **복사 → 새 탭에 붙여넣기 → 통째 저장** 방식으로 교체. 선택 종류와
  무관하게 선택한 것만 담긴다(글자 선택·개체 선택 모두 실측 확인).
- **표 안을 클릭만 하고 캡처하면 엉뚱한 것이 선택되던 문제.** 예전 자동 선택은
  표에서 빠져나온 뒤 옆 글자를 선택하는 방식이라, 복잡한 문서에서는 커서가
  앵커 옆이 아닌 곳에 떨어져 줄바꿈만 선택됐다. 커서가 속한 컨트롤을 직접
  얻어(`ParentCtrl`) 선택하는 방식으로 교체 — 위치 계산이 아예 없다.

### 알려진 제한

- **템플릿 빈칸에 서식 미리 지정**(개선안 29)은 미구현. 빈칸 자리의 서식이 실제로
  물려지는지 한글에서 실측한 뒤 설계할 항목이다 — 물려진다면 서식 감싸기
  문법으로 이미 충분할 수 있다.
- 서식 감싸기는 **중첩 미지원**이고, 템플릿 빈칸을 채우는 값 안에서는 아직 쓸 수
  없다.
- 되돌리기(Ctrl+Z) 횟수는 코드 구조에서 추론한 안내이며 실측하지 않았다.
- **한글에서 확인해야 할 것 2가지** (코드로는 판단할 수 없어 방어 코드만 넣어둠):
  1. `find_text`(RepeatFind)가 문서 끝에 닿으면 맨 앞으로 되감는가? 되감는다면
     앞쪽 `\` 가 지워질 수 있어, 삽입 지점보다 앞이면 멈추는 방어를 넣어 뒀다.
  2. 서식 감싸기가 실제로 감싼 부분에만 적용되는가? 서식 물림 문제는 논리적으로
     막아 뒀으나 실제 한글에서 확인하지 못했다.
- 양식 변환은 다른 줄과 섞어 쓸 수 없다 — 섞이면 변환을 막고 안내한다.
