## A. 지금 실제로 문제가 되는 것 (버그)

### 1. ✅🔴 라이브러리에서 삭제해도 팔레트 블럭이 고아로 남음 [실측]
템플릿 '결재란'을 라이브러리에서 지워도 팔레트 블럭은 그대로 남고, 클릭하면
"템플릿을 찾을 수 없습니다" 오류. 조각 파일은 이미 지워진 뒤라 복구 불가.
→ 삭제 시 참조하는 블럭도 같이 정리하거나, 최소한 경고("팔레트 2곳에서 사용 중").

### 2. ✅🔴 템플릿 이름을 바꾸면 팔레트 블럭이 깨짐 [추정]
블럭이 템플릿을 **이름 문자열**로 참조(`{"template": "결재란"}`). 이름 변경 시
연결이 끊긴다. → 이름 대신 **고유 id**로 참조해야 근본 해결.

### 3. 🔴 라이브러리 항목 이름 중복 처리와 팔레트 참조 충돌 [추정]
같은 이름 등록 시 `결재란 (2)`로 자동 변경되는데, 팔레트 블럭은 원래 이름을
들고 있어 엉뚱한 걸 가리킬 수 있다. (2번과 같은 뿌리)

### 4. 🔴 예외를 조용히 삼키는 곳이 23군데 [실측]
`except Exception: pass` — hwp_engine 12곳, palette_ui 3곳 등. 실패해도 사용자는
모르고 "왜 안 되지?"만 남는다. 오늘 겪은 버그들이 오래 숨어있던 이유.
→ 최소한 로그로 남기거나, 사용자에게 알릴 것과 무시할 것을 구분.

### 5. 🔴 `\` 청소가 문서 끝까지 훑어 무관한 역슬래시까지 지울 위험 [추정]
`strip_slot_markers(anchor)`가 anchor부터 **문서 끝까지** `\`를 지운다. 템플릿
아래에 사용자가 쓴 `\`가 있으면 같이 사라진다. → 삽입 범위 안으로 제한 필요.

### 6. 🔴 조각 삽입 시 원본 문서의 글꼴/스타일이 대상 문서로 딸려 올 수 있음 [추정]
`insert_file(keep_charshape=1, keep_style=1)` 기본값. 용지 여백은 안 따라오는 걸
확인했지만(실측), 스타일 오염 가능성은 미검증.

## B. 사용자를 헷갈리게 하는 것 (UX)

### 7. ✅🟠 "저장은 라이브러리, 배치는 환경설정" — 창이 갈라져 있음
템플릿 하나 쓰려면 창 2개, 단계 8개. 실제로 "템플릿 저장 어디 갔지?"를 겪음.
→ 환경설정의 `+템플릿`에서 **바로 캡처** 가능하게. (가장 큰 개선)

### 8. ✅🟠 등록할 때 이름·라벨·분류 3칸을 매번 물어봄
라벨은 99% 이름과 같고, 분류는 처음엔 불필요. `\계획서표지\` 사태의 원인.
→ **이름만** 묻고, 라벨·분류는 "▸자세히"로 접기.

### 9. ✅🟠 라이브러리 항목을 수정할 수 없음
등록하면 삭제 후 재등록뿐. 팔레트 블럭은 더블클릭으로 고쳐지는데 라이브러리는
안 되니 일관성도 없다. → 더블클릭 편집 + 템플릿 "다시 캡처".

### 10. 🟠 용어가 겹쳐서 헷갈림
라이브러리/템플릿/블럭/서식/기능/양식/팔레트/탭. 특히 **"서식"(라이브러리)과
"기능"(팔레트)이 사실상 같은 것**. → 통합하거나 이름을 갈라야.

### 11. ✅🟠 `\`가 두 가지 뜻 (빈칸 / 호출)
"감싸면 호출, 안 감싸면 빈칸" 규칙을 외워야 한다. 실제로 사용자가 `/`로 잘못
입력한 사건 발생. → 등록창에 **미리보기**("이렇게 쓰세요: `\보기\`")를 보여줄 것.

### 12. 🟠 실패해도 왜 실패했는지 안 알려줌
"기능은 글자를 선택한 뒤 눌러주세요" 정도는 있지만, 대부분은 상태표시줄에
한 줄 스치고 끝. → 실패 사유를 명확히.

### 13. 🟠 드래그 배치가 "타일 위에 놓기"만 됨
빈 칸에 직접 놓을 수 없고, 드래그 중 미리보기(어디 놓일지)가 없다.
사용자가 원한 "빈 공간에 블럭을 하나씩" 은 미구현.

### 14. 🟠 블럭이 많아지면 격자가 창 밖으로 넘침
스크롤이 없다. 빠른입력 25개 + 추가하면 화면을 벗어난다.

### 15. 🟠 팔레트 블럭 이름이 12자에서 잘림
`▦ 계획서표지…` 처럼 잘려서 뭐가 뭔지 구분이 안 될 수 있다. → 툴팁 필요.

### 16. 🟠 실행 취소(Undo)에 대한 안내가 없음
템플릿을 잘못 꽂았을 때 한글의 Ctrl+Z로 되돌아가는지 불명확. → 확인 후 안내.

## C. 구조·유지보수

### 17. ✅🟡 죽은 코드 4개 [실측]
`run_action`, `insert_marked_choice`, `set_markpen`, `apply_reset_format` — 호출 0곳.
원문자/형광펜 팔레트를 걷어내며 남은 잔해. → 삭제.

### 18. ✅🟡 `quickbuttons_ui.py`(112줄)가 통째로 미사용 [실측]
빠른입력이 팔레트 탭으로 흡수되며 아무도 import 안 함. → 삭제.

### 19. 🟡 `hwp_engine.py`가 986줄로 비대
연결·텍스트·서식·표·시험문제·캡처·변환이 한 파일에. → 최소 3개로 분리
(`engine_core` / `engine_exam` / `engine_library`).

### 20. ✅🟡 시험문제 전용 코드가 엔진 절반을 차지
`insert_bogi_box`, `insert_material_box`, `insert_photo_box`, `insert_experiment_box`,
`insert_question`, `_insert_choices` — 프로그램이 '팔레트'로 성격이 바뀐 지금,
이것만 별도 모듈로 빼는 게 맞다.

### 21. 🟡 설정이 두 군데로 갈라짐
`settings.py`(양식 프리셋)와 `palette.py`(탭·기본서식)가 같은 config.json을 쓰지만
API가 따로. → 하나로 정리하거나 역할을 명확히.

### 22. 🟡 테스트가 하나도 없음
`parser.build_library_plan` 같은 순수 함수는 한글 없이도 테스트 가능한데 없다.
→ 최소한 파서·라이브러리·팔레트 모델에 단위 테스트.

### 23. 🟡 매직넘버가 코드에 흩어져 있음
`_CELL_SIDE_MARGIN_MM = 3.6`, `0x2FFF1`, `range(200)`, `12자 자르기` 등.

### 24. 🟡 `import`가 함수 안에 들어간 곳들
`_diag`, `connect`, `_add_template` 등에서 지역 import. 의도(선택적 의존성)는
이해되나 일관성이 없다.

### 25. 🟡 버전 문자열이 실제와 안 맞음
`VERSION = "1.3.0"`인데 그 뒤로 라이브러리·팔레트·마크다운 변환이 통째로 들어옴.
docstring의 버전 이력도 v1.3.0에서 멈춤. → 정리 필요(사용자 지시 시).

## D. 기능 확장

### 26. ✅🔵 양식(.hwp 파일) 등록 — 완료
파일을 골라 등록 → 클릭하면 **새 문서로 열기**. 표지·통신문처럼 "이 양식으로
시작"하는 경우에 필요. 삽입은 페이지 설정이 안 따라오므로 별도 종류가 맞다.

### 27. 🔵 서식 감싸기 문법 (`\굵게\내용\/`)
현재 `\라벨\`은 문자·템플릿만. 서식은 팔레트 클릭으로만 적용 가능.
닫는 표시를 정해야 함(빈칸 `\`와 충돌하지 않게 `\/` 등).

### 28. 🔵 자간 자동 조절로 어절 단위 줄바꿈 — 미검토
사용자 요청. 한글에 "한글 단위/어절 단위 줄나눔" 문단 설정이 있는지,
자간 자동 조절이 API로 되는지 확인 필요.

### 29. 🔵 템플릿 빈칸에 서식 미리 지정
"이 빈칸에 들어갈 글자는 굵게 11pt" 를 템플릿에 심어두기.
현재는 빈칸 위치의 서식을 그대로 물려받는다. → 실제로 되는지 확인 필요.

### 30. 🔵 라이브러리 내보내기/가져오기
양식 프리셋에는 있는데(`export_profile`) 라이브러리엔 없다.
폴더째 복사는 되지만, 동료와 **항목 단위 공유**가 안 된다. README에
"폴더째 복사하면 이식 가능"이라 써뒀으니 정식 기능으로 만들 가치 있음.
